# 概述

MLP Cooker 提供完整的 RESTful API 接口，所有接口都以 `/api/` 开头。

## 认证

大部分 API 需要通过认证才能访问。认证方式：

1. 通过登录接口获取 token
2. 在后续请求的 Header 中携带认证信息

## 统一响应格式

所有 API 响应都遵循统一的格式：

### 成功响应

```json
{
  "success": true,
  "data": {...},
  "message": "操作成功"
}
```

### 错误响应

```json
{
  "success": false,
  "message": "错误信息",
  "error": {
    "error": "详细错误"
  }
}
```

## API 模块

API 接口按功能模块分类，详细文档请查看以下各模块：

### 基础模块

- [认证相关](./API-认证) - 用户登录、登出、获取用户信息等
- [配置管理](./API-配置管理) - 系统配置的查询、更新、验证等

### 资源管理模块

- [数据集管理](./API-数据集管理) - 数据集的增删改查和版本管理
- [模型管理](./API-模型管理) - 模型的增删改查和版本管理
- [镜像管理](./API-镜像管理) - 镜像的增删改查和版本管理
- [计算资源](./API-计算资源) - 队列、资源池等计算资源管理

### 任务执行模块

- [训练任务](./API-训练任务) - 训练任务的创建、查询、管理和监控

- [服务部署](./API-服务部署) - 服务的创建、查询、状态管理

### 应用模块

- [应用模板](./API-应用模板) - 应用模板的查询和管理

## 注意事项

1. **认证要求**：除了登录、登出、验证码和健康检查接口外，其他所有接口都需要认证
2. **请求方法**：大部分查询接口使用 `GET` 方法，创建使用 `POST`，更新使用 `PUT`，删除使用 `DELETE`
3. **分页参数**：列表查询接口通常支持 `pageNumber` 和 `pageSize` 参数
4. **路径参数**：使用 `:paramName` 格式表示路径参数，如 `:jobId`、`:datasetId` 等
5. **响应格式**：所有接口都遵循统一的响应格式，包含 `success`、`data` 和 `message` 字段
