Argo Workflows 管理

Argo Workflows 是一个用于 Kubernetes 的开源容器原生工作流引擎。本 API 提供了对 Argo Workflows 的完整管理功能。

所有 Argo Workflows 接口都需要认证。

接口列表​

服务信息​

  • GET /api/argo/info：获取 Argo 服务器信息
  • GET /api/argo/test-connection：测试 Argo 连接

Workflow 管理​

  • GET /api/argo/workflows：列出 Workflows
  • GET /api/argo/workflows/:workflowName：获取 Workflow 详情
  • POST /api/argo/workflows：创建 Workflow
  • DELETE /api/argo/workflows/:workflowName：删除 Workflow
  • PUT /api/argo/workflows/:workflowName/stop：停止 Workflow
  • PUT /api/argo/workflows/:workflowName/resubmit：重新提交 Workflow
  • PUT /api/argo/workflows/:workflowName/retry：重试 Workflow
  • GET /api/argo/workflows/:workflowName/logs：获取 Workflow 日志

WorkflowTemplate 管理​

  • GET /api/argo/workflow-templates：列出 WorkflowTemplates
  • GET /api/argo/workflow-templates/:templateName：获取 WorkflowTemplate 详情
  • POST /api/argo/workflow-templates：创建 WorkflowTemplate
  • DELETE /api/argo/workflow-templates/:templateName：删除 WorkflowTemplate
  • POST /api/argo/workflow-templates/:templateName/submit：从模板提交 Workflow

CronWorkflow 管理​

  • GET /api/argo/cron-workflows：列出 CronWorkflows
  • GET /api/argo/cron-workflows/:cronWorkflowName：获取 CronWorkflow 详情
  • POST /api/argo/cron-workflows：创建 CronWorkflow
  • DELETE /api/argo/cron-workflows/:cronWorkflowName：删除 CronWorkflow
  • PUT /api/argo/cron-workflows/:cronWorkflowName/suspend：暂停 CronWorkflow
  • PUT /api/argo/cron-workflows/:cronWorkflowName/resume：恢复 CronWorkflow

归档 Workflow 管理​

  • GET /api/argo/archived-workflows：列出归档的 Workflows
  • GET /api/argo/archived-workflows/:uid：获取归档的 Workflow 详情
  • DELETE /api/argo/archived-workflows/:uid：删除归档的 Workflow

配置说明​

在 config.yaml 中配置 Argo Workflows 连接信息：

ARGO_BASE_URL: http://localhost:2746
ARGO_NAMESPACE: argo
ARGO_TOKEN: your_argo_token

配置项说明：

  • ARGO_BASE_URL：Argo Server 地址
  • ARGO_NAMESPACE：默认命名空间
  • ARGO_TOKEN：访问令牌（可选，如果 Argo Server 需要认证）

接口详情​

GET /api/argo/info​

获取 Argo 服务器信息，包括版本和用户信息。

响应示例：

{
  "success": true,
  "data": {
    "version": {
      "version": "v3.5.0",
      "buildDate": "2024-01-15T10:00:00Z"
    },
    "userInfo": {
      "subject": "system:serviceaccount:argo:argo-server"
    },
    "config": {
      "baseURL": "http://localhost:2746",
      "namespace": "argo"
    }
  }
}

GET /api/argo/workflows​

列出 Workflows。

查询参数：

  • namespace：命名空间（可选）
  • labelSelector：标签筛选器（可选）
  • fieldSelector：字段筛选器（可选）
  • limit：返回数量限制（可选）
  • continue：分页令牌（可选）

响应示例：

{
  "success": true,
  "data": {
    "items": [
      {
        "metadata": {
          "name": "hello-world-xxxxx",
          "namespace": "argo",
          "creationTimestamp": "2024-01-15T10:00:00Z"
        },
        "status": {
          "phase": "Succeeded",
          "startedAt": "2024-01-15T10:00:00Z",
          "finishedAt": "2024-01-15T10:05:00Z"
        }
      }
    ]
  }
}

GET /api/argo/workflows/:workflowName​

获取指定 Workflow 的详情。

路径参数：

  • workflowName：Workflow 名称

查询参数：

  • namespace：命名空间（可选）

响应示例：

{
  "success": true,
  "data": {
    "metadata": {
      "name": "hello-world-xxxxx",
      "namespace": "argo"
    },
    "spec": {
      "entrypoint": "whalesay",
      "templates": [...]
    },
    "status": {
      "phase": "Succeeded",
      "nodes": {...}
    }
  }
}

POST /api/argo/workflows​

创建（提交）一个新的 Workflow。

请求体：

{
  "namespace": "argo",
  "serverDryRun": false,
  "workflow": {
    "apiVersion": "argoproj.io/v1alpha1",
    "kind": "Workflow",
    "metadata": {
      "generateName": "hello-world-"
    },
    "spec": {
      "entrypoint": "whalesay",
      "templates": [
        {
          "name": "whalesay",
          "container": {
            "image": "docker/whalesay:latest",
            "command": ["cowsay"],
            "args": ["hello world"]
          }
        }
      ]
    }
  }
}

响应示例：

{
  "success": true,
  "data": {
    "metadata": {
      "name": "hello-world-xxxxx",
      "namespace": "argo"
    }
  },
  "message": "Workflow 创建成功"
}

DELETE /api/argo/workflows/:workflowName​

删除指定的 Workflow。

路径参数：

  • workflowName：Workflow 名称

查询参数：

  • namespace：命名空间（可选）

响应示例：

{
  "success": true,
  "message": "Workflow 删除成功"
}

PUT /api/argo/workflows/:workflowName/stop​

停止正在运行的 Workflow。

路径参数：

  • workflowName：Workflow 名称

查询参数：

  • namespace：命名空间（可选）

响应示例：

{
  "success": true,
  "message": "Workflow 停止成功"
}

GET /api/argo/workflows/:workflowName/logs​

获取 Workflow 的日志。

路径参数：

  • workflowName：Workflow 名称

查询参数：

  • namespace：命名空间（可选）
  • podName：Pod 名称（可选）
  • containerName：容器名称（可选）

响应示例：

{
  "success": true,
  "data": {
    "logs": "hello world\n..."
  }
}

POST /api/argo/workflow-templates/:templateName/submit​

从 WorkflowTemplate 提交新的 Workflow。

路径参数：

  • templateName：WorkflowTemplate 名称

请求体：

{
  "namespace": "argo",
  "parameters": {
    "message": "hello world",
    "count": "3"
  }
}

响应示例：

{
  "success": true,
  "data": {
    "metadata": {
      "name": "hello-world-xxxxx",
      "namespace": "argo"
    }
  },
  "message": "Workflow 提交成功"
}

CronWorkflow 管理​

CronWorkflow 用于定时执行 Workflow，类似于 Kubernetes 的 CronJob。

POST /api/argo/cron-workflows​

创建 CronWorkflow。

请求体：

{
  "namespace": "argo",
  "cronWorkflow": {
    "apiVersion": "argoproj.io/v1alpha1",
    "kind": "CronWorkflow",
    "metadata": {
      "name": "hello-world-cron"
    },
    "spec": {
      "schedule": "0 0 * * *",
      "workflowSpec": {
        "entrypoint": "whalesay",
        "templates": [...]
      }
    }
  }
}

PUT /api/argo/cron-workflows/:cronWorkflowName/suspend​

暂停 CronWorkflow 的定时执行。

PUT /api/argo/cron-workflows/:cronWorkflowName/resume​

恢复 CronWorkflow 的定时执行。

错误处理​

所有接口遵循统一的错误响应格式：

{
  "success": false,
  "message": "错误描述",
  "error": "详细错误信息"
}

常见错误码：

  • 401：认证失败，Token 无效
  • 404：Workflow 不存在
  • 500：Argo Server 内部错误

使用建议​

  1. 命名空间管理：建议为不同的项目使用不同的命名空间
  2. 标签使用：为 Workflow 添加标签，便于筛选和管理
  3. 日志查询：定期清理旧的 Workflow 日志
  4. 模板复用：使用 WorkflowTemplate 提高 Workflow 的可重用性
  5. 定时任务：使用 CronWorkflow 实现定时执行的任务
