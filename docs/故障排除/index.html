故障排除

本章节提供常见问题的解决方案。

前端问题​

依赖安装失败​

问题：运行 npm install 时出错

解决方案：

  1. 检查 Node.js 版本是否 >= 20.0.0
  2. 删除 node_modules 和 package-lock.json
  3. 清除 npm 缓存：npm cache clean --force
  4. 重新安装：npm install

端口被占用​

问题：启动时提示端口 8000 被占用

解决方案：

  1. 查找占用端口的进程：lsof -i :8000
  2. 终止进程：kill -9 <PID>
  3. 或修改配置使用其他端口

页面白屏​

问题：访问页面显示白屏

解决方案：

  1. 检查浏览器控制台是否有错误
  2. 确认后端服务是否正常运行
  3. 检查代理配置是否正确
  4. 清除浏览器缓存

后端问题​

配置文件错误​

问题：启动时提示配置文件错误

解决方案：

  1. 确保 config.yaml 文件存在于项目根目录
  2. 检查 YAML 格式是否正确
  3. 确认所有必需的配置项都已填写

TypeScript 编译错误​

问题：运行 npm run build 时出错

解决方案：

  1. 检查 TypeScript 版本
  2. 删除 dist 目录
  3. 重新编译：npm run build

端口被占用​

问题：启动时提示端口 8001 被占用

解决方案：

  1. 查找占用端口的进程：lsof -i :8001
  2. 终止进程：kill -9 <PID>

API 调用问题​

CORS 错误​

问题：前端调用 API 时出现 CORS 错误

解决方案：

  1. 检查后端 CORS 配置
  2. 确认前端代理配置正确
  3. 检查请求 URL 是否正确

认证失败​

问题：API 返回 401 未认证错误

解决方案：

  1. 确认已正确登录
  2. 检查 AK/SK 是否有效
  3. 清除浏览器缓存重新登录

网络连接错误​

问题：无法连接到 AIHC 平台 API

解决方案：

  1. 检查网络连接
  2. 确认 ML_PLATFORM_RESOURCE_BASE_URL 配置正确
  3. 检查防火墙设置
  4. 查看后端日志获取详细错误信息

登录问题​

AK/SK 验证失败​

问题：登录时提示 AK/SK 无效

解决方案：

  1. 确认输入的 AK/SK 正确
  2. 检查 AK/SK 是否有访问权限
  3. 联系管理员确认账号状态

配置未保存​

问题：登录成功但配置未保存

解决方案：

  1. 检查 config.yaml 文件权限
  2. 确认后端有写入权限
  3. 查看后端日志确认错误原因

获取帮助​

如果以上方案无法解决您的问题,请：

  1. 查看后端日志：backend/logs/access.log
  2. 在 GitHub 提交 Issue
  3. 联系技术支持
